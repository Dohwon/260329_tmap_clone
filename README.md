# Tmap Clone Web

React + Vite 기반의 TMAP 클론 PWA입니다. 실제 TMAP 경로, 검색, 주변 POI, 주행 기록, 고속도로/국도 중심 드라이버 UX를 목표로 개발 중입니다.

## Structure

- `server.js`: Railway 배포용 Express 서버와 TMAP/연료 API 프록시
- `src/store/appStore.js`: 전역 상태, 경로 탐색, 안내, 저장 경로 재개
- `src/services/tmapService.js`: TMAP 검색/경로/주변 POI 연동
- `src/components/Map`: 지도, 도로/카메라/속도 레이어
- `src/components/Navigation`: 안내 배너, 실시간 재탐색, 근처 주유소/휴게소/주차장
- `src/components/Route`: 경로 카드, 경유지, 프리셋, 미리보기

## 0413 Changes

- 더보기의 `안전 운전 모드`는 홈 이동 메뉴가 아니라 실제 on/off 토글 스위치로 분리했고, 스위치 버튼만 상태를 바꾸도록 수정
- 일반도로 세그먼트는 제한속도/평균속도 표시가 과도하게 70~100km/h로 튀지 않도록 도로 유형별 상한과 기본값을 보정
- 안내 중 운전자 추종은 기기 heading 대신 `현재 위치 앞의 실제 경로 벡터` 기준으로 회전시키고, 차량이 화면 중심에 더 가깝게 유지되도록 카메라 오프셋을 줄임
- 안내 중에는 무거운 오버레이를 줄여 회전 추종 반응성을 높였고, 차선 준비 패널은 다음 안내가 약해도 최대한 유지되도록 보강
- 안내 중 검색을 열 수 있게 변경했고, 검색 결과 선택 시 `경유지 추가` 또는 `목적지 변경`으로 분기되게 수정
- 안내 화면에서 검색 진입점이 두 군데 뜨던 문제를 정리하고 상단 안내 영역만 남김
- 안내 시작 직후 하단 `현재 경로 유지` 패널이 자동으로 내려가 지도 시야를 가리지 않게 했고, 위로 스와이프하거나 하단 영역을 터치해 다시 펼칠 수 있게 수정
- 안내 중 주유소 목록에서 `근방 최저`와 `경로상 최저`를 바로 경유지로 선택할 수 있게 수정
- 드라이브 기록 저장은 실제 주행 polyline 기준으로 판단하고, 동일한 실제 코스는 재저장 프롬프트를 띄우지 않도록 수정
- 움직임 없이 안내를 종료한 경우에는 `이동 없음` 기록으로 구분되도록 정리
- 초보/중수/고수 프리셋이 기본 추천 경로 순서를 뒤집지 않도록 ETA 우선 정렬로 조정
- 분기점 옵션은 `어디로 연결되는지`가 먼저 보이도록 문구와 카드 구조를 보정
- 차로 준비 문구는 상단 파란 배너에서 제거하고, 초록색 `차선 준비` 영역에만 유지하도록 정리
- 검색 입력 삭제나 API 실패 후 화면이 죽던 문제를 수정
- 국도 데이터에 `restStops`가 없는 경우에도 검색이 크래시하지 않도록 방어 로직 추가
- `주유소/휴게소/주차장` 근처 검색의 실패 내성을 높였고, 휴게소는 고속도로 정적 데이터로 폴백되도록 추가
- `강남역`, `양화대교`, `올림픽대로` 같은 자주 쓰는 검색어는 즉시 후보를 보여주도록 빠른 후보/캐시 로직 보강
- 실제 주행 후 저장된 경로를 그대로 다시 안내할 수 있도록 저장 경로 재개 로직 추가
- 저장 경로는 안내 종료 시점에 실제 주행 궤적이 있으면 실제 주행 polyline 기준으로 저장되도록 수정
- 안내 중 프리셋을 `초보/중수/고수`로 다시 바꿀 수 있게 하고, 저장 경로는 실시간 재탐색 없이 원본 경로를 유지하도록 보정
- 경로 세그먼트별 제한속도/평균속도 정보를 지도에 노출할 수 있도록 TMAP 응답 파서를 확장
- 실제 `segmentStats.averageSpeed/speedLimit/congestionScore`가 있는 링크만 혼잡도 오버레이로 강조하고, 정체/서행 구간에 실속 팝업을 추가
- 저장된 실제 주행 경로를 안내 경로와 비교해 `경로 이탈`, `실제 선호 우회`, `급감속 구간`을 분석하고 더보기의 드라이브 기록/운전 습관 화면에 노출
- `guideLane/laneInfo` 패턴 사전을 확장해 `1~2차로 이용`, `우측 2개 차로 유지` 같은 차선 힌트를 실제 안내 문구 우선으로 파싱
- 검색/안내 회귀를 검증하기 위해 `scripts/navigation-harness.mjs`에 검색 크래시 방지 테스트를 추가
- `scripts/navigation-harness.mjs`에 차선 힌트 파싱과 실주행 편차/급감속 분석 검증을 추가
- 아직 남아 있는 지도/차선/실시간 재탐색 한계는 `docs/known-limitations.md`에 계속 기록

## 0414 Changes

- `DATA_GO_KR_API_KEY` 기반 병원 영업정보 보강을 정리했고, Railway에서 신규 경로가 씹히던 문제를 피해 기존 정상 동작 메타 엔드포인트(`/api/meta/tmap-status`)를 통해 병원 메타를 우회 조회하도록 변경
- `GOOGLE_PLACES_API_KEY` 기반 음식점 보강을 추가해 TMAP 음식점 후보에 대해 Google 평점, 리뷰 수, 영업중 여부를 붙일 수 있게 구성
- 음식점은 `TMAP 후보 생성 -> Google Places 보강` 순서로 처리하고, 매칭 실패 시 `별점 정보 없음`으로 안전하게 폴백하도록 수정
- 검색 결과에서 음식점을 누르면 바로 길찾기로 보내지 않고 상세 카드가 먼저 열리도록 바꿨고, 카드에서 `경유지 추가` 또는 `목적지 변경`을 선택할 수 있게 구성
- 사용자가 음식점에 직접 평점을 남길 수 있는 저장소를 추가했고, 현재 위치가 해당 지점 반경 250m 안에서 `20분 이상 체류`했을 때만 평점 입력이 열리도록 제한

## 0414 Scenic / Nav Stabilization

- 안내 시작 직후 `navAutoFollow`를 effect 내부에서 반복 갱신하던 루프를 제거해 React `#185` 계열 크래시를 막음
- route `polyline`, `segmentStats`, `junctions`, `cameras`, `maneuvers`에 좌표 정규화/필터링을 넣어 malformed TMAP 응답으로 지도/안내 화면이 죽지 않도록 보강
- 경관도로 선택은 더 이상 임시 경로 시간만 늘리는 방식이 아니라 `실제 경유지 추가 -> 재탐색` 흐름으로 변경
- 경관 추천은 현재 선택된 메인 경로선의 복사본(`scenicReferencePolyline`)을 기준으로 계산해, 이미 추가된 경유지 때문에 경로 기준선이 흔들리지 않도록 수정
- 경관 추천은 `원본 경로 앞쪽`, `원본 경로 기준 80km 이내`, `원본 경로선 반경 20km 이내` 후보만 남기고, 역주행에 가까운 후보는 빨간 경고를 띄운 뒤 뒤쪽으로 정렬
- 경관 추천 카드에는 `진입 위치` 주소를 붙여 사용자가 실제 어디로 들어가는지 바로 비교할 수 있게 함
- 홈의 고속도로/국도 카드와 고속도로 탐색 시트에도 `시점 주소 → 종점 주소`를 작은 글씨로 함께 표기
- `음식점` 주변 검색 fallback이 일반 TMAP POI 400을 반복해서 치지 않도록 경로맛집/로컬 fallback 우선으로 정리

## Current Open Tasks

- 차선 위치 표출:
  - 현재는 `차선 준비` 문구와 단순 차로 패턴까지만 제공하며, 지도 위 실제 차선 위치/포켓차선/직좌 분기 형상 표출은 미완료

  - 현재는 운전자 추종 회전과 전방 오프셋을 적용했지만, TMAP 수준의 부드러운 운전자 기준 회전 지도와 차선 레벨 진입 시점 표현은 아직 부족

  - Google Places 매칭 실패, 키 미설정, API 제한, 동일 상호 다중지점 오매칭 상황에서는 `별점 정보 없음`으로 남는다
- 실제 기반 안전/카메라 정밀도:
  - 카메라, 구간단속, 제한속도는 아직 모든 구간에서 실데이터 완전 일치 수준이 아니므로 추가 정제 필요
- 실시간 재탐색 안정성:
  - `/api/tmap/routes` 400/429, `nearestRoad` 403 계열 회귀는 하네스와 실서비스 로그 기준으로 계속 검증 필요
