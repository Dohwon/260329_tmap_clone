# MVP Forward Roadmap 2026-04-16

이 문서는 장애 복구 문서와 별개로, 우리 MVP 기준에서 앞으로 해야 할 일을 실행 태스크 형식으로 정리한다.

우리 MVP의 중심은 아래 세 가지다.

- 좋은 길을 찾는다
- 좋은 길을 실제로 달리게 안내한다
- 왜 그 길이 좋은지 실제 데이터로 설명한다

즉 `좋은 도로 추천`과 `실제 주행 안내`가 중심이고, 맛집/병원/부가정보는 그 다음이다.

## 기준

- `우선순위`는 MVP 가치 기준이다.
- `시급성`은 지금 MVP 체감을 얼마나 좌우하는지 기준이다.
- `선행`은 먼저 끝나야 다음 태스크를 진행할 수 있는 작업이다.
- `후행`은 이 작업 뒤에 자연스럽게 이어지는 작업이다.
- `작업`은 최대 2개 중심으로 적는다.
- `검증`은 실제 사용 관점에서 확인할 수 있게 적는다.

## 1. 좋은 길을 실제로 추천하기

### 1-A. 안전/카메라 actual-only 정합성

- 우선순위: `P0`
- 시급성: `치명적`
- 선행: `open-task 1-C`
- 후행: `1-B`
- 작업:
  - 경로 추천 카드의 카메라 수를 live route `safetyFacilityList` 기준으로만 계산
  - 도로 탐색 카메라 synthetic fallback을 전 도로에서 제거하고 actual-only 기준으로 정리
- 검증:
  - 추천 카드의 카메라 수와 실제 경로 응답이 크게 어긋나지 않음

### 1-B. 고속도로/국도/경관 master data 확장

- 우선순위: `P0`
- 시급성: `높음`
- 선행: `1-A`
- 후행: `1-C`
- 작업:
  - 한국도로공사/민자도로 데이터를 road master에 병합
  - 휴게소, 졸음쉼터, IC/JC, 카메라, scenic entry point 누락 노선을 계속 보강
- 검증:
  - 서해안, 서울양양 등 대표 노선에서 누락 체감이 줄어듦

### 1-C. 좋은 도로 추천의 실제 waypoint화

- 우선순위: `P0`
- 시급성: `치명적`
- 선행: `1-B`, `open-task 1-A`
- 후행: `1-D`
- 작업:
  - 해안도로/산악도로/고속도로 추천 시 road snap waypoint를 실제 state에 저장
  - 추천 카드에 진입 위치 주소와 지명을 고정해서 “어디로 들어가는 길인지” 바로 보이게 함
- 검증:
  - 추천 도로 선택 후 ETA만 늘고 waypoint가 안 생기는 현상이 사라짐

### 1-D. 경로 앞쪽 우선 추천과 역주행 경고

- 우선순위: `P0`
- 시급성: `높음`
- 선행: `1-C`
- 후행: `3-A`
- 작업:
  - 원본 경로선 기준 앞쪽 후보를 우선 추천하고 되돌이표/순환 추천을 차단
  - 역주행 1회 허용은 유지하되 경고를 더 명확히 표시
- 검증:
  - 서해안고속도로 주행 중 경로와 무관한 뒤쪽/옆쪽 후보가 먼저 뜨지 않음

## 2. 추천한 길을 실제로 따라가게 하기

### 2-A. 실제 경로 추종 안정화

- 우선순위: `P0`
- 시급성: `치명적`
- 선행: `open-task 1-A`, `open-task 1-B`, `open-task 2-A`
- 후행: `2-B`, `2-C`
- 작업:
  - route 400/403/429와 GPS jump로 인해 추종이 끊기는 구간을 줄이고, 안내 시작 시 malformed route를 안전 차단
  - 지나온 길 삭제와 남은 경로 trim을 실제 matched progress 기준으로 고정
- 검증:
  - 안내 시작 후 경로가 흰 화면 없이 유지되고, 지나간 구간이 즉시 잘림

### 2-B. 운전자 시점 안내 강화

- 우선순위: `P1`
- 시급성: `높음`
- 선행: `2-A`
- 후행: `2-C`
- 작업:
  - 운전자 추종 상태머신을 고정하고 회전 직전 미니 인셋을 추가
  - 주요 분기점에서 “다음 어디로 연결되는지”를 더 크게 강조
- 검증:
  - 첫 회전/합류에서 바로 다음 안내가 우선 보이고 길을 놓치지 않음

### 2-C. 차선 힌트 고도화

- 우선순위: `P1`
- 시급성: `중간`
- 선행: `2-B`
- 후행: `4-A`
- 작업:
  - EDC 기준 `laneTurnInfo`, `etc=64/65/90`, 포켓차선, 직좌/직우 패턴 사전을 확장
  - 상단 배너 대신 별도 차선 영역과 인셋 중심으로 차선 힌트를 정리
- 검증:
  - 직좌, 버스전용, 포켓차선 케이스에서 차선 문구가 더 구체화됨

## 3. 좋은 길이라는 이유를 실제 데이터로 설명하기

### 3-A. 안전/경관/도로 정보 설명 카드

- 우선순위: `P1`
- 시급성: `높음`
- 선행: `1-D`, `2-A`
- 후행: `3-B`
- 작업:
  - 추천 경로 카드에 카메라, 단속, 도로 유형, scenic 진입 정보가 실제 데이터 기준으로 붙게 정리
  - “왜 이 길인지”를 preset 차이보다 먼저 설명하게 구성
- 검증:
  - 사용자가 추천 경로를 볼 때 길의 성격과 이유를 바로 이해할 수 있음

### 3-B. 초보/중수/고수의 실제 차별화

- 우선순위: `P1`
- 시급성: `중간`
- 선행: `2-A`, `3-A`
- 후행: `3-C`
- 작업:
  - 초보는 복잡 합류 회피, 중수는 균형, 고수는 속도 우선으로 경로 차이를 실제 데이터 기준으로 반영
  - 경로 설명 카드에 왜 차이가 나는지 짧게 표시
- 검증:
  - 같은 출발지/목적지에서도 preset별 차이가 논리적으로 설명 가능

### 3-C. 실제 주행 기록과 추천 품질 피드백 연결

- 우선순위: `P2`
- 시급성: `중간`
- 선행: `3-B`, `open-task 2-C`
- 후행: 없음
- 작업:
  - 저장된 실제 주행 기록을 바탕으로 사용자가 자주 선택한 우회/회피 패턴을 추천 로직에 반영
  - “실제로는 이 길을 더 자주 탔다”는 피드백을 다음 추천에 연결
- 검증:
  - 같은 사용자 반복 주행에서 추천 경로가 실제 선택 패턴을 일부 반영함

## 4. MVP 이후 보강 요소

### 4-A. 주행 맥락 정보

- 우선순위: `P2`
- 시급성: `중간`
- 선행: `1-B`, `2-C`
- 후행: `4-B`
- 작업:
  - 경로상 주유소/휴게소 actual-only 정렬과 할인 적용가/경로상 최저 선택 액션을 고정
  - 경로상 맛집 후보와 지도 중심 맛집 탐색을 분리
- 검증:
  - 주행 중 필요한 보조 정보가 메인 경로 추천을 방해하지 않음

### 4-B. 병원/맛집/부가 정보 정리

- 우선순위: `P2`
- 시급성: `낮음`
- 선행: `4-A`
- 후행: 없음
- 작업:
  - 병원 영업시간, 음식점 Google 평점/리뷰/영업중 메타를 안정화
  - 비음식점 평점 노출 재회귀를 계속 막음
- 검증:
  - 부가 정보가 “있으면 좋음” 수준으로 안정 제공됨

근거: 좋은 길 추천 자체를 실제 데이터로 만들기 위한 핵심이다.

2. `2-A`, `2-B`
근거: 추천한 길을 실제로 따라가게 하는 단계다.

근거: 왜 이 길이 좋은지 설명하고 운전자 성향 차이를 만들기 위한 단계다.

근거: 핵심 추천/안내 이후의 고도화와 보강 요소다.
