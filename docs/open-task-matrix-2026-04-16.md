# Open Task Matrix 2026-04-16

이 문서는 현재 남아 있는 미완료 항목만 정리한다. 완료된 항목은 의도적으로 제외했고, 각 카테고리 안에서 선행/후행 관계가 보이도록 태스크를 다시 잘랐다.

## 기준

- 상태값은 `P0`, `P1`, `P2`로 나눈다.
- `시급성`은 지금 당장 사용자 경험을 깨는 정도다.
- `우선순위`는 `장애 복구 + MVP 가치`를 함께 반영한다.
- `선행`은 먼저 끝나야 다음 태스크를 안전하게 진행할 수 있는 작업이다.
- `후행`은 해당 태스크가 끝나야 착수 가능한 작업이다.
- `검증`은 하네스 또는 수동 확인 기준이다.

## 1. 실시간 TMAP 경로 적용 / 직선거리 금지 / 안내 시작 시 실제 경로

### 1-A. `/api/tmap/routes` 400 원인 고정

- 우선순위: `P0`
- 시급성: `치명적`
- 선행: 없음
- 후행: `1-C`, `1-D`, `5-A`
- 작업:
  - 서버 프록시에서 빈 `waypoints`, 잘못된 `origin/destination`, 중복 좌표를 요청 전에 제거
  - 검색 입력 삭제, 경유지 추가, 안내 중 목적지 변경, scenic waypoint 추가 시의 요청 본문을 각각 분리 로깅
  - 400 응답 body를 dev에서 바로 볼 수 있게 에러 구조를 정규화
- 검증:
  - `강남역`, `양화대교`, `올림픽대로`, 경유지 2개 추가, scenic 추가 조합으로 `scripts/navigation-harness.mjs` 재현
  - 같은 시나리오에서 dev/prod 모두 400 미발생

### 1-B. `nearestRoad` 403 회귀 제거

- 우선순위: `P0`
- 시급성: `치명적`
- 선행: 없음
- 후행: `1-D`, `2-A`
- 작업:
  - `nearestRoad` 호출을 `안내 시작 직전/이탈 판단/출발지 수동 지정`에만 제한
  - 스냅 실패 시 경로 요청 자체를 막지 않고 raw 좌표 또는 route polyline projection으로 폴백
  - 403이 반복되면 일정 시간 `nearestRoad` 재시도를 차단하는 circuit breaker 추가
- 검증:
  - dev에서 `nearestRoad 403`가 떠도 안내 시작이 중단되지 않음
  - 스냅 실패 후에도 실제 route polyline은 유지됨

### 1-C. 429 방지용 라우트 요청 예산 제어

- 우선순위: `P0`
- 시급성: `치명적`
- 선행: `1-A`
- 후행: `6-A`, `7-A`, `9-B`
- 작업:
  - 동일 origin/destination/waypoint 조합은 TTL 캐시 강제
  - inflight route 요청 중복 제거
  - 버튼 연타, 검색 debounce, preset change, off-route reroute를 endpoint budget 단위로 제한
  - 사용량 초과 시 `즉시 실패`가 아니라 마지막 성공 route를 재사용하고 재시도 시점을 명시
- 검증:
  - 경로 1회 요청 시 네트워크 탭에서 route 요청이 1~2회 수준으로 제한
  - 429 발생 시 화면은 죽지 않고 마지막 정상 경로를 유지

### 1-D. 안내 시작 경로 유효성 게이트

- 우선순위: `P0`
- 시급성: `치명적`
- 선행: `1-A`, `1-B`
- 후행: `3-A`, `9-A`

## 즉시 개선안

- route/nearestRoad/hazard/fuel을 엔드포인트별 예산으로 나눠서 관리한다. 지금은 기능별 상태가 아니라 API별 예산 통제가 더 중요하다.
- `idle`, `screen hidden`, `mode off`, `route panel hidden` 상태에서는 background API를 기본 중지한다.
- 429나 quota 에러는 바로 재시도하지 말고 `마지막 성공값 유지 + 재시도 가능 시각 표시`로 바꾼다.
- `nearestRoad`는 모든 위치 샘플에 붙이지 말고 `안내 시작`, `재탐색 직전`, `출발지 수동 지정`에서만 제한 호출한다.
- Google Places, TTS, TMAP route는 전부 inflight dedupe와 disk/cache TTL을 강제해 같은 입력에 재과금되지 않게 한다.
- 경로 추종과 경로 생성이 정상화되기 전에는 `초보/중수/고수` 차이 고도화 작업을 보류한다. 지금은 프리셋보다 기본 route fidelity가 우선이다.

## 확장 아이디어

- 사용자 액션 기반 `API heatmap`을 만들어 어떤 버튼이 어떤 엔드포인트를 몇 번 치는지 화면 단위로 기록한다.
- `quota safe mode`를 추가해 free tier가 임계치에 가까우면 Google 평점/병원 보강/배경 hazard를 자동 단계 하향한다.
- route preview는 `actual route core`, `optional enrichment`를 분리해서 핵심 길찾기와 부가 정보가 서로를 망치지 않게 한다.
- 도로 master data는 `TMAP live route`, `한국도로공사`, `수동 검증 로그` 3개 계층으로 버전 관리한다.
